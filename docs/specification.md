# 📋 Техническое задание (ТЗ)

**Проект:** Мобильное ПО для помощи слабовидящим и незрячим  
**Рабочее название:** EchoSight / «Голосовой ассистент для слабовидящих»  
**Версия документа:** 1.1  
**Дата составления:** 2026-03-04

> **Контекст:** кроссплатформенное мобильное приложение (Android + iOS) на React Native + TypeScript, основной способ взаимодействия — голос (ASR и TTS), без внешнего оборудования. Языки: русский и английский. Команда: 6 человек. Бюджет: ~5 000 ₽. Срок MVP: 4 месяца.

---

## 1. Краткое описание задачи

Создать мобильное приложение (MVP) для помощи слабовидящим и незрячим гражданам во взаимодействии с окружающим миром:

- **Чтение** печатного текста
- **Распознавание** предметов/сцены
- **Определение** денежных купюр
- **Голосовое управление** и озвучивание

**Фокус:** надёжность, простота использования, работа офлайн.

## 4. Область покрытия MVP

> Приоритизация по **MoSCoW**

### ✅ Must-have — без них MVP не имеет ценности

#### 4.1. Голосовой интерфейс

- Активация одним нажатием (кнопка ≥ 48×48 dp, центр экрана) или встряхиванием (shake-to-activate)
- ASR (русский/английский) для команд: **«Прочитай»**, **«Опиши»** / **«Что это»**, **«Помощь»**, **«Повтори»**, **«Стоп»**
- TTS для всех ответов; скорость 0.5×–2.0×, настройка громкости
- Звуковая обратная связь: тон при начале записи, тон при результате, вибрация при ошибке

#### 4.2. OCR (чтение печатного текста)

- Офлайн-распознавание (русский + английский) через ML Kit
- Пользователь говорит «Прочитай» → захват кадра → распознавание → озвучивание
- Подсказка при плохом качестве: _«Текст нечёткий — поднесите камеру ближе»_

#### 4.3. Детекция объектов

- Распознавание бытовых объектов (COCO + дообученные классы, см. Приложение A)
- Озвучивание топ-3 по confidence; порог ≥ 0.45
- При отсутствии детекций: _«Не удалось распознать объекты — направьте камеру иначе»_

#### 4.4. Интерфейс доступности

- Полная совместимость с **TalkBack** (Android) и **VoiceOver** (iOS)
- Accessibility-атрибуты (`accessibilityRole`, `accessibilityLabel`) на каждом компоненте; touch-target ≥ 48×48 dp
- Поддержка масштабирования шрифтов и высокой контрастности

#### 4.5. Настройки и история

- Хранение: язык, голос TTS, скорость речи
- Локальная история последних 50 распознаваний в SQLite (без сохранения изображений)
- «Повтори» — озвучить последний результат

#### 4.6. Управление энергопотреблением

- Камера и ML-модели активируются **только по запросу**
- Нет фонового сканирования
- Авто-деактивация камеры через 30 с бездействия

### 🟡 Should-have — реализуются при наличии времени (Sprint 3–4)

#### 4.7. Определение денежных купюр

- Классификатор MobileNet v3-Small (fine-tuned): ₽ (50–5000), $ (1–100)
- Датасет: Open Images + собственная выборка ≥ 200 фото/номинал
- Если не готов — скрывается за **feature flag**

#### 4.8. Голосовой туториал (onboarding)

- Пошаговый голосовой гид при первом запуске (≤ 2 мин)
- Повторный вызов командой «Помощь»

### 🔵 Could-have — отложены на post-MVP

- **4.9.** Описание расположения объектов (лево/центр/право)
- **4.10.** Навигационные подсказки (GPS + POI)
- **4.11.** Wake-word активация без нажатия кнопки

### ⛔ Won't-have в MVP

- Распознавание лиц, генеративные описания сцен (LLM), live-видеопомощь

## 5. Функциональные требования

### 5.1. Голосовые сценарии

#### Общий flow обработки команды

```
Нажатие / встряхивание → [тон начала] → ASR (≤ 5 с) → Command Processor
  → не распознано: «Не понял — скажите ещё раз» + [вибрация]
  → распознано: «Распознано: <команда>» → модуль → [тон результата] → TTS
```

#### Сценарий «Прочитай»

1. Захват кадра (autofocus, max 1280×960)
2. Preprocessing: автобаланс яркости/контраста через утилиты предобработки
3. ML Kit Text Recognition → извлечение блоков текста
4. Если текст пустой / confidence < 0.4 → _«Текст не обнаружен — поднесите камеру ближе»_
5. Иначе: TTS зачитывает блоки слева→направо, сверху→вниз
6. Команда «Стоп» прерывает озвучивание

#### Сценарий «Опиши» / «Что это»

1. Захват кадра → resize до 300×300 (SSD MobileNet input)
2. TFLite inference через JSI (react-native-fast-tflite)
3. NMS (IoU = 0.45), confidence ≥ 0.45
4. Результат: _«Вижу: бутылка, книга»_ (топ-3). Если 0 детекций — _«Не удалось распознать объекты»_

#### Сценарий «Купюра» *(Should-have, feature flag)*

1. Захват кадра → crop центральной области 60% → resize 224×224
2. MobileNet v3-Small → softmax → top-1
3. Confidence < 0.6 → _«Не уверен — поднесите купюру ближе»_
4. Иначе: _«Купюра: 500 рублей»_

#### Сценарий «Помощь»

Озвучить список доступных команд с краткими пояснениями.

#### Сценарий «Повтори»

Повторно озвучить последний результат из SQLite-истории.

### 5.2. Дополнительные требования

- Все результаты дублируются **вибрацией** (короткая — успех, двойная — ошибка)
- При длительной операции (> 1.5 с) — промежуточное сообщение _«Обрабатываю…»_
- Многоязычность: язык ASR/TTS через настройки; UI-строки через `i18next` + `react-i18next` (JSON locale-файлы, RU/EN)
- Отклонение разрешений камеры/микрофона → объяснение + предложение открыть системные настройки

### 5.3. Ограничения и допущения

- **Не реализуются в MVP:** распознавание лиц, LLM-описания, live-видеопомощь, GPS-навигация
- Приложение **не передаёт** изображения/аудио на серверы без явного согласия
- OCR оптимизирован под **печатный текст** (рукописный — вне scope)
- Макс. размер изображения: 4096×4096 px (downscale если больше)

**Vosk** (on-device, Apache-2.0) через `react-native-vosk`:

- Модели: `vosk-model-small-ru` (~45 МБ), `vosk-model-small-en-us` (~40 МБ)

- Fallback: если модель не загружена → инструкция + предложение скачать

> *Альтернатива (оценена, отложена):* PocketSphinx — хуже качество RU, нет React Native-плагина.

### 7.3. TTS

- **Основной:** системный TTS через `react-native-tts` (Android `TextToSpeech`, iOS `AVSpeechSynthesizer`)
- **Опционально (post-MVP):** RHVoice (LGPL-3.0) — расширенный русский голос (~1 неделя)
- Настройки: скорость 0.5×–2.0×, громкость, выбор голоса

### 7.4. OCR

**Google ML Kit On-device Text Recognition v2** (`@react-native-ml-kit/text-recognition`):

- Кириллица + латиница офлайн; модель ~15 МБ
- Preprocessing: автокоррекция яркости/контраста через утилиты `imagePreprocessing.ts`

**Fallback:** Tesseract (react-native Tesseract-обёртка) для Huawei без GMS (~12 МБ/язык, ручная загрузка traineddata).

### 7.5. Computer Vision (детекция объектов)

**TensorFlow Lite** через `react-native-fast-tflite` (JSI-based):

| Модель | Описание |
|--------|---------|
| SSD MobileNet v2 (COCO) | 80 классов, ~4 МБ, INT8. Покрывает ~60% объектов из Приложения A |
| Дообученная модель | Transfer learning для ключей, кошелька, дорожных знаков RU. Обучение через Colab. |
| MobileNet v3-Small | Классификатор купюр (fine-tuned) |

- **Делегаты:** NNAPI (Android), Core ML (iOS) — ~2× ускорение
- **Квантование:** post-training INT8 для всех моделей

### 8.2. Описание компонентов

| Компонент | Описание |
|-----------|---------|
| **UI Layer** | Минимум экранов: главный (кнопка + статус), настройки, история. Все компоненты с `accessibilityRole` / `accessibilityLabel`. |
| **Voice Interface** | Vosk ASR через `react-native-vosk`. Нативный модуль с JS-биндингами. |
| **Command Processor** | Центральный оркестратор (Zustand store). Парсинг по ключевым словам (regexp) → dispatch → TTS. Error Boundary при исключениях. |
| **OCR Module** | ML Kit Text Recognition v2 через `@react-native-ml-kit/text-recognition`. Preprocessing через утилиты. |
| **Object Detector** | TFLite inference через `react-native-fast-tflite` (JSI). NMS + фильтрация в TypeScript. |
| **Banknote Classifier** | Аналогично Object Detector, другая модель и crop-стратегия. Защищён feature flag. |
| **Camera Service** | Singleton на `react-native-vision-camera`: `initialize` → `capture` → `release`. Авто-отключение через 30 с. |
| **TTS Module** | Обёртка `react-native-tts`; FIFO-очередь; «Стоп» очищает очередь. |
| **Storage Module** | SQLite (`react-native-sqlite-storage`): `settings` (key-value), `history` (id, type, text, timestamp). `react-native-keychain` для чувствительных данных. |
| **Local Logger** | Structured JSON-логи (уровень, timestamp, component, message). Без PII. Ротация 5 МБ, 3 файла. |

### 8.3. Жизненный цикл ML-моделей

| Этап | Поведение |
|------|----------|
| **Lazy init** | Модели загружаются в память только при первом вызове команды |
| **Кеширование** | TFLite model / TextRecognizer хранится в памяти до `dispose` или `AppState === 'background'` (> 60 с) |
| **Выгрузка** | Системное предупреждение о памяти → принудительный dispose; при следующем запросе — повторная загрузка |

**Потребление RAM:**

| Модель | Память |
|--------|--------|
| Vosk ASR (нативный модуль) | ~80 МБ |
| TFLite SSD MobileNet v2 (INT8) | ~20 МБ |
| TFLite Banknote classifier | ~10 МБ |
| ML Kit OCR (управляется платформой) | ~40 МБ |
| **Итого пиковое** | **~150 МБ** |

> На устройстве с 3 ГБ RAM — приемлемо.

### 8.4. Обработка ошибок (Error Boundary)

Единый контракт всех модулей:

```typescript
type Result<T> = Success<T> | Failure
// Success: { ok: true, data: T }
// Failure: { ok: false, errorCode: string, userMessage: string }
```

При `Failure` Command Processor:
1. Логирует ошибку (Local Logger)
2. Озвучивает `userMessage` через TTS
3. При критической ошибке → предлагает действие

**Типичные ошибки:**

| Ситуация | Fallback-сообщение |
|----------|--------------------|
| Камера занята | _«Камера используется — закройте другое приложение»_ |
| Модель не загружена | _«Нужно скачать языковой пакет. Подключитесь к интернету»_ |
| ASR тишина (timeout 5 с) | _«Не услышал команду — нажмите кнопку и скажите ещё раз»_ |

### 8.5. Потоки данных и многопоточность

| Поток | Содержимое |
|-------|-----------|
| **JS Thread** | UI (React), Command Processor (Zustand), TTS, Camera Service |
| **Native ASR Thread** | Vosk recognition loop. Возвращает текст через event bridge |
| **JSI / Native Thread** | TFLite inference через `react-native-fast-tflite` (JSI-биндинг, синхронный вызов из JS). Object Detector / Banknote поочерёдно |
| **Native Modules** | Разовые CPU-задачи (image preprocessing, ML Kit OCR) |

> JSI обеспечивает прямой вызов нативного кода из JS без Bridge overhead.

### 8.6. Протоколы и данные

- Все вызовы — **локальные**, в памяти
- JS ↔ Native: JSI-биндинги (`react-native-fast-tflite`, `react-native-vision-camera`), Native Modules (`react-native-vosk`, `@react-native-ml-kit/text-recognition`)
- Потенциально REST/HTTPS для облака в будущем

## 11. Тестирование и валидация

### 11.1. Тестовые сценарии

| Модуль | Что тестируется |
|--------|----------------|
| ASR | Обработка команд в тихой и шумной обстановке |
| OCR | Читаемость стандартных печатных текстов (разные шрифты/контраст) |
| CV | Распознавание объектов при разном освещении и расстоянии |
| Купюры | Определение номинала в различных положениях и при разном освещении |
| Регрессия | Тесты на сохранённой тестовой выборке |

### 11.2. Пилотное тестирование

- **10–15 реальных пользователей** из ЦА
- Метрики: SUS, NPS, удовлетворённость
- Обязательное тестирование с TalkBack/VoiceOver

### 11.3. Метрики для мониторинга

- Accuracy CV / OCR (общая и по категориям)
- Latency (время от команды до результата)
- Количество ошибок (неопознанные/ложноположительные)
- Показатели использования (частота функций)
- Пользовательские метрики (SUS, NPS)

#### Группа 2 — Требуют дообучения (transfer learning)

| Класс | Источник данных |
|-------|----------------|

| Кошелёк | Open Images V7 + собственная выборка ≥ 300 изображений |
| Коробка | Open Images V7 + собственная выборка ≥ 300 изображений |

| Ступенька / бордюр | Собственная выборка |

**Итого классов MVP: ~28.** Список уточняется после Sprint 0.

> ⚠️ Объекты безопасности (ступенька, traffic light, stop sign) имеют **пониженный** confidence-порог **0.35** (vs 0.45 для остальных).
